# Инструкция по деплою

## Текущая ситуация

Ваш проект настроен на **автоматический деплой через GitHub Actions**. Это означает, что при каждом push в ветку `main` проект автоматически собирается и деплоится.

## Как задеплоить сейчас

### Способ 1: Автоматический деплой (текущий)

1. Закоммитьте изменения:

    ```bash
    git add .
    git commit -m "Ваше сообщение"
    git push origin main
    ```

2. GitHub Actions автоматически запустит деплой
3. Проверьте статус в разделе `Actions` на GitHub
4. После успешного деплоя сайт будет доступен

### Способ 2: Ручной деплой через gh-pages

Если хотите использовать ручной деплой, нужно изменить настройки:

1. **Измените настройки GitHub Pages:**
    - Перейдите на GitHub: `Settings` → `Pages`
    - В разделе `Build and deployment` → `Source`
    - Измените с `GitHub Actions` на `Deploy from a branch`
    - Выберите ветку: `gh-pages`
    - Папка: `/ (root)`
    - Нажмите `Save`

2. **Выполните деплой:**

    ```bash
    npm run deploy
    ```

3. **Проверьте результат:**
    - Ветка `gh-pages` будет создана/обновлена
    - GitHub Pages задеплоит содержимое через 1-2 минуты

## Почему ручной деплой не работает сейчас?

GitHub Pages настроен на GitHub Actions как источник. Когда вы запускаете `npm run deploy`, он создает/обновляет ветку `gh-pages`, но GitHub Pages не использует эту ветку, потому что настроен на GitHub Actions.

## Рекомендация

**Оставьте автоматический деплой через GitHub Actions** — это проще и надежнее:

- Не нужно вручную запускать команды
- Автоматическая сборка при каждом push
- Меньше вероятность ошибок

Если есть проблемы с автоматическим деплоем (как сейчас с npm ci), их нужно исправить в workflow файле, а не переключаться на ручной деплой.
